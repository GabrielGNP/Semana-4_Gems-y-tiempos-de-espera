Demo: Implicit Wait — Capacidades y Limitaciones

Caso 1: Elemento que aún no existe

En este caso, el test debe pasar debido a que selenium:

1. Buscará el botón Iniciar Caso 1 y lo accionará

2. Buscará repetitivamente el elemento nuevo hasta que aparezca en el DOM o pasen 10 segundos

3. El elemento aparecerá en el DOM después de 3 segundos

4. Una vez que el elemento esté presente, el test continuará y encontrará el elemento.

WebElement elemento = driver.findElement(By.id("delayed-element"));
...
assertNotNull(elemento); assertEquals("¡Aparecí después de 3 segundos!", elemento.getText());

Al presionar el botón, un elemento aparecerá en el DOM después de 3 segundos:

Implicit wait debería esperar y encontrarlo. ✅

Caso 2: Elemento oculto

En este caso, el test NO debe pasar debido a que selenium:

1. El botón SI existe en el DOM, pero tiene el estilo display: none.
Esto hace que el botón esté inaccesible para un usuario normal

2. Selenium lo buscará y lo encontrará.

3. Intentará realizar las acciones como un usuario normal.

4. Fallará debido a que el elemento no es accesible.

WebElement botonOculto = driver.findElement(By.id("hidden-btn"));
...
botonOculto.click();

Elementos testeados

El botón existe en el DOM pero tiene display:none. No se puede hacer clic. ❌

Caso 3: Texto que cambia después

En este caso, el test NO debe pasar debido a que:

1. El elemento SI existe en el DOM desde el principio y tiene un texto

2. Selenium presiona el botón Iniciar Caso 3.

3. Selenium buscará el elemento y lo encontrará.

4. Recuperará el contenido del elemento (texto del elemento).

5. El elemento cambiará de texto después de 3 segundos.

6. Selenium arrojará un error porque el texto que recuperó es distinto al esperado
el texto esperado es el texto que aparece luego de 3 segundos.

WebElement elemento = driver.findElement(By.id("changing-text"));
String textoObtenido = elemento.getText();
...
assertEquals("Texto actualizado", textoObtenido, "Implicit wait NO espera cambios de texto. Se obtuvo: '" + textoObtenido + "'");

Elementos testeados

Texto inicial

Al presionar el botón, el texto cambiará a "Texto actualizado" después de 3 segundos. ❌

Caso 4: Elemento deshabilitado

En este caso, el test NO debe pasar debido a que:

1. El elemento SI existe en el DOM desde el principio y está deshabilitado

2. Selenium buscará el elemento y lo encontrará.

3. Intentará utilizarlo (por ejemplo, escribir en el campo).

4. Selenium arrojará un error porque el elemento está deshabilitado.

WebElement inputDeshabilitado = driver.findElement(By.id("disabled-input"));
inputDeshabilitado.sendKeys("texto de prueba");
String valorActual = inputDeshabilitado.getAttribute("value");
...
assertEquals("texto de prueba", valorActual, "No se pudo escribir en el campo deshabilitado. Implicit wait NO espera habilitación.");

Elementos testeados

El input existe pero está disabled. No se puede escribir en él. ❌
